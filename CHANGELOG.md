# История изменений

## 2026-06-28 (безопасность)

- **Роли на сервере**: в Edge Function `save-store` менеджеру разрешено только добавлять новые записи; правка/удаление существующих и удаление товаров — только админ (`mergeStore(..., isAdmin)`). UI больше не единственная защита.
- **Достоверный журнал**: `store_changes` заполняется из реальной серверной разницы (`describeChange`), а не из текста клиента — подделать нельзя.
- **Лимит тела** функции `save-store` считается по реальным байтам (`readJsonCapped`), а не по `content-length`.
- **Очистка фото**: осиротевшие `.webp` удаляются из bucket `product-photos` при сохранении.
- **CSP**: убран `'unsafe-inline'` для `script-src`; весь UI переведён с inline-`onclick` на делегирование событий (`data-act` + реестры в `app.js`).
- **id записей** теперь `newId()` (мс·1000 + random) — без коллизий при оффлайн-создании на разных устройствах.
- `escapeHtml` экранирует и одинарную кавычку.
- **CORS** обеих Edge Functions ограничен своими источниками (`*.vercel.app`/`localhost`, переопределение секретом `ALLOWED_ORIGINS`) вместо `*`.

## 2026-06-09 (v1.1.0)

- Длинные комментарии в таблице расходов теперь переносятся по словам (`overflow-wrap`), а не обрезаются.
- Таблица склада ("Анбор") сделана компактной (`compactTable`) для маленьких экранов: подобраны ширины колонок, уменьшено фото товара.
- Кнопки действий в компактных таблицах подстраиваются под текст (перенос строк, минимальная высота для удобного нажатия).
- Добавлены адаптивные стили для очень узких экранов (до 380px): статистика, периоды, навигация и таблицы истории.
- Изменения отправлены в GitHub: `4e56a34 Improve mobile table layout and wrap long comments`.

## 2026-05-28

- Добавлены явные `GRANT` в `supabase-schema.sql` для таблиц `store_state`, `user_roles`, `store_changes`, чтобы проект был готов к новым правилам Supabase Data API.
- Перед изменениями Supabase был сделан SQL-экспорт данных.
- Усилена Edge Function `create-user`: теперь при ошибке записи роли новый Auth-пользователь не остается без роли, а ошибка возвращается понятнее.
- Edge Function `create-user` задеплоена в Supabase.
- Удалены служебные файлы Windows `desktop.ini`; добавлено правило `desktop.ini` в `.gitignore`.
- В таблицу "История прихода" добавлена колонка "Сумма" (`кол-во * цена`).
- Добавлен простой логотип `Magazin` на страницу входа и в шапку приложения.
- Таблицы истории операций сделаны компактнее для экранов ноутбуков и телефонов.
- Проверена локальная сборка через `npm run build`.
- Изменения закоммичены и отправлены в GitHub: `3f80ff0 Update store app and Supabase access`.
